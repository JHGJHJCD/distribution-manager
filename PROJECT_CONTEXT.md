# PROJECT_CONTEXT.md — הקשר מלא לשיחה חדשה

## רקע ושיחה קודמת
האפליקציה נבנתה מ-0 בשיחה אחת. המשתמש ניתח קובץ Excel קיים (`מערכת חלוקה.xlsx`) ובקש לבנות אפליקציה Windows שעושה אותו דבר בצורה נוחה יותר. הקובץ המקורי כולל 10 גיליונות (הוראות שימוש, מקבלים, חלוקה השבוע, חד פעמי, מעקב חלוקות, עדכון קבוצתי, טיוטה למעקב, חיפוש מהיר, סיכום, בקשות לשיפור).

## העדפות המשתמש שנאספו בשאלות בירור
- **מיון** רשימת חלוקה: אלפביתי (לא לפי דחיפות)
- **חלוקה הבאה**: מחושבת לפי תדירות — שבועי: רביעי הבא, דו-שבועי: רביעי בעוד ~2 שבועות, חודשי: רביעי בעוד ~30 יום, חד-פעמי/ריק: רביעי הקרוב
- **חד-פעמי שלא הגיע**: נחשב כמי שקיבל (נשאר בתחתית הרשימה)
- **עדכון קבוצתי** → מעדכן אוטומטית "חלוקה אחרונה" לכל מי שסומן ✓
- **גיבוי**: בכל שמירה, לתיקייה שנבחרת ידנית
- **היסטוריית שינויים**: רק שינויי סטטוס (מושהה/הסתיים)
- **הדפסה**: Portrait, מיון אלפביתי, עמודות: שם + כל טלפונים + ✓ ביצוע

## אלגוריתם חד-פעמיים (one_time.py)
1. משתמש מזין כמה מוצרים זמינים (`products_spin`)
2. `compute_suggested_n(total)` → מחסר קבועים פעילים → N לחד-פעמיים
3. רשימה ממוינת: **הכי ישן קודם**, tie-break: **יותר נפשות קודם**
4. N ראשונים מסומנים אוטומטית (checkbox) — אפשר לשנות ידנית
5. כפתור "הוסף לעדכון קבוצתי" → מוסיף ל-`GroupUpdateTab._rows_data` + שומר ב-`_extra_ids`
6. אחרי חלוקה → מקבל חד-פעמי יורד לתחתית (last_distribution מתעדכן)

## תיקונים שבוצעו (לאחר בדיקת GPT-5.5)
1. ייבוא Excel: חיפוש גיליון "מקבלים" + זיהוי שורת כותרת לפי "שם"
2. FK `ON DELETE SET NULL` + מחיקה חוסמת אם יש היסטוריה (ValueError)
3. `bulk_add_distributions` שומר `notes` + מחשב `next_dist` לפי תדירות
4. דיאלוג מקבל כולל שדות QDateEdit לתאריכים
5. חד-פעמיים מוסרו מרשימה השבועית
6. `compute_suggested_n` מחזיר tuple `(n, regular_count)`
7. `_extra_ids` ב-GroupUpdateTab שומר חד-פעמיים בין רענונים
8. export נשמר ב-`exports/` בתיקיית האפליקציה
9. גורד ב-`update_recipient` כש-dict ריק
